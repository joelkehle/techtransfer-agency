Key Design Points

  • All agent communication goes through the bus — no direct agent-to-agent calls
  • The Operator is an agent — it registers with the bus and sends messages like any other agent
  • PDF Extractor is one binary deployed as multiple instances, configured by -next-agent-id and -capability flags at deploy time
  • Subject matter expert agents can be contributed by different teams — they only need pkg/busclient and a capability name
  • Each agent authenticates with its own HMAC-SHA256 secret — the bus and each agent share a key that signs every request, proving identity without transmitting passwords. HMAC verifies who sent a message and that it wasn't tampered with. It does not encrypt message content — agent-to-bus traffic runs on a private Docker network today; remote deployment would add TLS for encryption.
  • Capability names (patent-screen, prior-art-search) are the protocol contract — bus and agents agree on strings, nothing else
